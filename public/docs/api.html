Status API

Programmatic access to the same data the homepage shows. Aggregated from status.claude.com.

Endpoint

GET https://wasclaudedown.today/api/status

Query parameters

  • format — json or markdown. Optional; falls back to Accept header.
  • tz — IANA timezone (e.g. America/New_York) used to determine what “today” means. Defaults to UTC.

Content negotiation

  • Accept: application/json → JSON body
  • Accept: text/markdown → Markdown body
  • Anything else → plain text

Example

curl -H "Accept: application/json" \
  "https://wasclaudedown.today/api/status?tz=America/New_York"

Response (JSON)

{
  "answer": "NO",
  "checked_at": "2026-04-18T12:34:56.000Z",
  "current_status": { "level": "ok", "label": "All systems operational right now" },
  "currently_down": false,
  "down": false,
  "incidents": [],
  "reason": "Anthropic reports Claude as operational right now.",
  "timezone": "America/New_York"
}

Discovery

  • API catalog: /.well-known/api-catalog (RFC 9727)
  • OpenAPI spec: /docs/openapi.json
  • Agent skills: /.well-known/agent-skills/index.json
